## 循环不变量

理解二分查找中的循环不变量是关键。循环不变量是在循环的每次迭代开始前都成立的逻辑条件。

**关键点：循环不变量：目标值（若存在）必在 `[left, right)` 中**
以左开右闭区间为例：

* 初始 `left = 0`，`right = n`，保证所有有效索引 `0..n-1` 都在 `[0, n)` 内。
* 循环条件 `left < right` 等价于区间非空
* 保持：根据 `nums[mid]` 与 `target` 的比较，如果 `nums[mid] < target`，则 `mid` 及左侧都不可能是目标，`left` 应该排除 `mid`，即新区间为 `[mid + 1, right)`。此时保证了目标（若存在）仍在区间中；反之 `right` 应该排除 `mid`，即新区间为 `[left, mid)`。每次更新都在维持不变量的前提下尽可能缩小区间。
* 终止：当 `left == right` 时，区间 `[left, right)` 内无整数索引，循环结束。

## 不单独处理相等的情况

允许重复元素的情况，不单独处理相等的情况，可以查找找到**左边界**或**右边界**。
对于左闭右开区间写法：

* 寻找第一个等于 target 的位置
  * **策略**：相等时继续向左搜索，因此相等时移动**右边界**。
  * 最终 `right` 指向第一个 `>= target` 的位置
* 寻找最后一个等于 target 的位置（右边界）
  * **策略**：相等时继续向右搜索，因此相等时移动**左边界**。
  * 最终 `left` 指向最后一个 `<= target` 的位置的后一个位置，即指向第一个 `> target` 的位置

> 个人理解：相等时移动哪个边界，代表从这个边界排出，此时这个边界旁边就是最后一个满足条件的，而由于开闭区间情况是否包含这个边界不同，需不需要±1修正依具体情况而定

### 对比记忆

| 区间类型       | 找左边界（相等时） | 找右边界（相等时） |
| -------------- | ------------------ | ------------------ |
| 左闭右闭 \[l,r] | `r = mid - 1`      | `l = mid + 1`      |
| 左闭右开 \[l,r) | `r = mid`          | `l = mid + 1`      |
| 左开右开 (l,r) | `r = mid`          | `l = mid`          |
